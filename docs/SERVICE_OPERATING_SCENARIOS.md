# LogisticsLink Ocean Service Operating Scenarios

이 문서는 LogisticsLink Ocean을 실제 플랫폼 서비스로 운영할 때 확인해야 할 기본 시나리오를 정리한다. 로컬 또는 리허설 환경에서는 `npm run seed:scenario`로 아래 흐름을 검증할 수 있다.

## 운영 원칙

- 고객에게 보이는 표현은 `LogisticsLink`와 `LogisticsLink Ocean`을 사용한다.
- 시나리오 계정과 샘플 데이터는 로컬/리허설 검증용이다. 운영 환경에는 `npm run seed` 또는 `npm run seed:bootstrap`만 적용한다.
- 운임 기준, 입찰, 낙찰, 운송 상태 변경은 모두 권한과 감사 로그 기준으로 추적 가능한 흐름이어야 한다.

## 시나리오 1. 기업 온보딩

1. 신규 기업이 `/signup`에서 계정을 신청한다.
2. 이메일과 사업자번호 중복을 확인한다.
3. 계정은 승인 전 상태 안내를 본다.
4. 운영자가 `/admin`에서 계정 상태를 검토하고 활성화한다.
5. 활성화된 사용자는 역할별 화면에 진입한다.

성공 기준:

- 중복 이메일과 사업자번호가 차단된다.
- 비활성 계정은 견적, 풀, 입찰 API를 사용할 수 없다.
- 활성 계정은 역할에 맞는 화면으로 이동한다.

## 시나리오 2. 화물 수요 등록

1. 화주 또는 포워더가 `/shipper` 또는 `/forwarder`에서 화물 수요를 등록한다.
2. 필수 정보는 POL, POD, ETD, 운송 모드, 컨테이너, 수량, 중량이다.
3. 시스템은 화물 물량과 추천 공동구매 풀 후보를 계산한다.

성공 기준:

- 등록된 견적은 `logisticslink-ocean` 서비스 코드로 분리된다.
- 사용자는 본인 권한 범위 안의 견적만 볼 수 있다.
- 조건이 맞는 풀 후보가 추천된다.

## 시나리오 3. 블라인드 공동구매 풀

1. 사용자가 견적 기반으로 새 공동구매 풀을 만들거나 추천 풀에 참여한다.
2. 풀은 참여자의 개별 회사명과 견적 상세를 다른 참여자에게 공개하지 않는다.
3. 풀은 집계 수량, 항로, ETD, 화물 조건을 기준으로 운영된다.

성공 기준:

- 참여 기업의 정체성은 일반 참여자에게 노출되지 않는다.
- 풀의 물량 합계가 참여자 물량과 일치한다.
- 풀 상태는 집계 중 상태로 유지된다.

## 시나리오 4. 타임락 역경매

1. D-14 기준으로 운영자가 타임락 전환을 실행하거나 배치가 자동 실행된다.
2. 경매 가능 풀이 선사 입찰 보드에 표시된다.
3. 선사는 운임 기준 상한보다 낮고 현재 최저가보다 유효한 입찰을 제출한다.

성공 기준:

- 운임 기준 상한을 초과한 입찰은 차단된다.
- 현재 최저가보다 불리한 입찰은 차단된다.
- 입찰 보드는 최신 최저가를 반영한다.

## 시나리오 5. 낙찰 확정

1. 운영자가 `/admin`에서 경매를 마감한다.
2. 시스템은 최저 유효 입찰을 낙찰 후보로 판단한다.
3. 운영자는 낙찰 선사와 최종 운임을 확정한다.

성공 기준:

- 낙찰 선사와 최종 운임이 풀에 기록된다.
- 풀 상태는 낙찰 상태로 전환된다.
- 낙찰 이후 입찰은 추가로 받지 않는다.

## 시나리오 6. 운송 실행과 후속 관리

1. 운영자가 낙찰 건을 계약 또는 운송 중 상태로 전환한다.
2. 화주, 포워더, 선사는 권한 범위 안에서 운송 상태를 확인한다.
3. 운송이 완료되면 완료 상태로 전환한다.
4. 실패, 취소, 분쟁 상태가 필요한 경우 운영자가 예외 상태로 관리한다.

성공 기준:

- 상태 전환은 허용된 상태 흐름만 따른다.
- 제한 데이터는 권한 없는 역할에 노출되지 않는다.
- 운영자는 예외 상태와 후속 조치가 필요한 건을 식별할 수 있다.

## 시나리오 데이터

`npm run seed:scenario`는 다음 데이터를 만든다.

- 운영자: `admin@logisticslink.co.kr`
- 화주: `shipper@logisticslink.co.kr`
- 포워더: `forwarder@logisticslink.co.kr`
- 선사: `carrier@logisticslink.co.kr`
- 공통 비밀번호: `LogisticsLink!123`
- KRPUS to USLGB dry cargo 집계 풀
- KRPUS to USLAX reefer cargo 진행 중 경매와 선사 입찰
- SCFI, carrier FAK, public tariff, internal master 기준 운임

운영 환경에는 이 데이터를 넣지 않는다.
